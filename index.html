▶ Half-Sheet Video Modal

Single video embedded inside a standard half-sheet IAM. Copy, title, and CTA live in the modal layout — not overlaid on the video. Copy and screen zone guidelines from the Reels tab do not apply here.

📌 Because copy and CTA live outside the video frame in the modal's own layout, there are no character limits driven by video real estate. Write copy to fit the modal — not the video overlay constraints.

Content Handoff

For stakeholders, PMs, and creative teams submitting video content for any in-app format — Reels, half-sheet, or silent loop.

Before you record
Check Content Timing first to match your video length to the content type. Check Screen Zones if shooting for Reels — top and bottom 25% of frame are reserved for UI. Videos that don't meet spec need re-recording.

1

Export your video as MP4

All formats require MP4 (H.264). No MOV, no Loom links, no Google Drive previews — the raw file only. Match your export settings to the ratio table above.

2

Include a poster image (Reels only)

A static first-frame thumbnail is required for Reels so the card doesn't flash black on load. Not required for half-sheet or silent loop.

Format: PNG or JPG 1080 × 1920px Frame 0–1s of the video
3

Write your copy

Provide per-card copy. For Reels, follow the character limits (copy overlays the video). For half-sheet and silent loop, write to fit the modal layout — no strict limits.

FieldReels (overlay)Half-Sheet / Silent Loop
Headline3–6 words, benefit-led1–2 lines, no strict limit
Body1 sentence, 5–10 words1–3 sentences
CTA label2–3 words, verb-first2–4 words, verb-first
CTA destinationDeep link or URL — test in-app before handoff
4

Host on GitHub Pages and get your source URLs

GitHub Pages does not show file URLs anywhere — there is no link to copy. Build the URL manually using this formula:

https://[username].github.io/[repo-name]/[filename]

Enable Pages: Repository → Settings → Pages → Deploy from branch → main → Save. Takes ~1–2 min. Verify each URL in a browser before sending — URLs are case-sensitive.

Make sure the repository is set to Public. Private repos won't serve files via GitHub Pages and Braze won't be able to load them.
5
